UI / UX Design

Modern companies want to have an appealing website to get visitors, but schools are no exception. To develop the website for the College of Arts this fall, we focused on two parts - UX design and UI design. In the part of user interface, I coded in the prototyping with HTML by adding hierarchical menu, embedded link button, slideshow, etc. to improve the functionality; on the other hand, it is very important to improve the user experience, which required a consideration of usefulness, usability, and desirability. I applied Universal Principles of Design and AODA to design web pages with peers, such as selection of color, shape and size, and implemented Agile Method to interact with students and professors to get feedback. Finally, the website was improved and now the professors are satisfied with it. Therefore, it will be a great opportunity to work on UI and UX design for companies in the future.

Architectural Pattern

Architectural structure is important to an application as it is a blueprint. MVC framwork is what we used in the application, m stands for model to manage business logic and data, v stands for view to present information and data, c stands for control to handle user interactions. In the level of middleware, by calling apis in the application, we can request for the action or data and get a response back to achieve certain functionalities. It is potentially to be an computer architecture as I go deeper down.

SDLC - Testing and Deployment Phase

As we are moving everything to the cloud, some legacy systems, struts2 and weblogic in my case, need a migration in order to be compatible with other systems. We used divide and conquer method to divide a large project into small modules, so we can do unit testing for each modules and once each modules is finished, we can do integration testing altogether. After testing is done, we found some bugs, and we go back and fix them, checked code in version control - CA Harvest SCM and process continuous integration and continuous delivery by using Jenkins. It is a good start to become a future QA tester.
