Architectural Pattern

Architectural pattern is a reusable solution to a commonly occuring problem in software architecture, which is similar to software design pattern but with a broader scope. MVC framework is what we used in the web application, m stands for model, which is to manage business logic and data, v stands for view, which is to present information and data, c stands for control, which is to handle user interactions. The advantage of using the mvc framework is its separate concerns, which separates business logic, interface, and data from each other, makes it easy to maintain for further developer. As the application integrates multiple applications across the enterprise, middleware acts as a hidden translation layer, requesting data from other applications by calling APIs, hence no direct communication between the two, and returning responses to implement certain functionality. It is possible to become a computer architect when I look more deeply into.

DevOps Process Flow

DevOps is a software development methodology that combines software development with IT operations, which is complementary to Agile software development. As we are moving everything to the cloud, legacy systems (in my case struts2 and weblogic) need to be migrated to be compatible with other systems, and we were processing devops. After the coding was done, we handled continuous testing with a divide-and-conquer approach - breaking the project into smaller modules to do unit testing and intergration testing repeatly. We then processed continuous integration by checking the code in version control each time and performed continuous delivery as code changes are automatically built, tested, and packaged for release into production by using Jenkins. This could be a good start to become a future DevOps Engineer on building automation of blockchain software or a QA tester to access software quality through manual and automated testing.
